## Bug Analysis

### Bug: Timer cycle doesn't continue after T4:0 (Green) completes

**Root causes identified:**

1. **`getAddressValue()` in ladderEngine.ts** — When address is `T4:0.DN`, it does `startsWith('T4:')` ✓ then `endsWith('.DN')` ✓, but looks up `memory.timers['T4:0.DN']` which doesn't exist. It needs to extract the base address `T4:0` from `T4:0.DN`.

2. **Shallow copy in `evaluateRungs()`** — `{ ...memory }` doesn't deep copy `timers`, `inputs`, `outputs`, `bits`. The mutations in `handleTimerTON` and `evaluateRung` mutate Zustand state directly, breaking reactivity.

3. **`SimulationLoop.tsx` stale closure** — `ladderMemory` is captured at render time, but `ladderTick(delta)` updates the store. The outputs read in the same frame are stale.
